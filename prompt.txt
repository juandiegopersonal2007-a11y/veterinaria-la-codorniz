Actúa como un equipo experto de desarrolladores web full-stack con más de 10 años de experiencia en seguridad web, diseño UX/UI y sistemas de gestión veterinaria.
Tu tarea es crear el código completo y funcional de una página web profesional y segura para una clínica veterinaria llamada "La Codorniz", ubicada en Tecomán, Colima, México. La página será desplegada en Netlify.

🔐 SEGURIDAD — PRIORIDAD MÁXIMA
Implementa las siguientes medidas de seguridad de forma obligatoria:
Autenticación del Admin:

Las credenciales de administrador NUNCA deben estar en texto plano en el código fuente.
Usa hashing con SHA-256 (Web Crypto API nativa del navegador) para almacenar y comparar contraseñas.
Guarda en localStorage únicamente el hash, jamás la contraseña real.
Implementa bloqueo de cuenta tras 5 intentos fallidos consecutivos (bloqueo de 15 minutos con contador en localStorage).
La sesión admin debe tener expiración automática de 30 minutos de inactividad.
Genera un token de sesión aleatorio con crypto.getRandomValues() al iniciar sesión, guárdalo en sessionStorage (no localStorage) para que se destruya al cerrar el navegador.

Protección de datos de mascotas:

Los datos de mascotas almacenados en localStorage deben estar cifrados con AES-GCM usando la Web Crypto API nativa.
La clave de cifrado se deriva del ID de la mascota + una salt fija usando PBKDF2.
Nunca exponer datos en texto plano en el DOM ni en la consola del navegador.

Seguridad general del sitio:

Agrega en el HTML las siguientes meta etiquetas de seguridad:

html<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline' https://cdnjs.cloudflare.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' data: https:; connect-src 'self'">
<meta http-equiv="X-Content-Type-Options" content="nosniff">
<meta http-equiv="Referrer-Policy" content="strict-origin-when-cross-origin">
```
- Crea un archivo `_headers` para Netlify con los siguientes headers HTTP de seguridad:
```
/*
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: camera=(), microphone=(), geolocation=()
  Strict-Transport-Security: max-age=31536000; includeSubDomains
  X-XSS-Protection: 1; mode=block
```
- Sanitiza **todos los inputs** del usuario antes de procesarlos para prevenir **XSS**.
- Escapa todo contenido dinámico antes de insertarlo en el DOM (usa `textContent` en lugar de `innerHTML` donde sea posible).
- El panel admin **no debe ser accesible por URL directa**; solo debe mostrarse tras autenticación exitosa con token válido en sessionStorage.
